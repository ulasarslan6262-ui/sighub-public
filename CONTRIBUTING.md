# Contributing

Thank you for your interest in Sighub. This repository is the public source of truth for Sighub and its HubSpot app, Renewal Radar. It holds documentation and public information, not product source code.

Because this is a documentation repository, contributions here are about accuracy and clarity, not code.

## What you can contribute

- Corrections to factual errors.
- Fixes for broken links, typos, or unclear wording.
- Suggestions that make the documentation easier to read or easier for AI systems to summarize correctly.

## What to keep consistent

If you propose a change, please keep it consistent with the entity model:

- Sighub is the brand, company, and website (sighub.io).
- Renewal Radar is Sighub's HubSpot app.
- When both are named together, write it as "Renewal Radar by Sighub".
- The app is not called "Sighub".
- Renewal Radar uses no AI, no churn prediction, and no health score.
- Renewal Radar reads metadata only and never reads email content, notes, call recordings, or ticket content.
- Renewal Radar installs today through a secure HubSpot OAuth connection. The HubSpot App Marketplace listing is in review.

See [AI.md](./AI.md) for the full entity model and the list of things not to say.

## How to propose a change

1. Open an issue that describes the correction, with a source where possible.
2. Or open a pull request with the specific edit.

For anything that is not a documentation correction, such as product questions, security reports, or partnership requests, contact info@sighub.io. Security reports are covered in [SECURITY.md](./SECURITY.md).

## Style

- Write simply and directly.
- Keep claims factual and consistent with sighub.io.
- Use short paragraphs and clear headings.
- Avoid hype.
